### Buckyball 是什么

Buckyball 是一个面向 DSA 的通用开源架构框架。DSA，也就是 Domain-Specific Architecture，指的是针对某一类特定计算任务专门优化的体系结构，例如为机器学习、图计算、信号处理或科学计算设计的专用加速器，比如加速Transformer的脉动阵列实现。Buckyball 的目标不是定义某一种固定的加速器，而是通过统一的架构抽象、接口规范和系统支持，让不同领域专用架构可以无感地集成到同一个系统中，并被高效调度、验证和运行。

### 为什么要这样一个框架？

很多时候，一个 DSA 的核心计算逻辑面积在整个芯片或FPGA上并不大，但真正落地时却需要处理指令接入、任务调度、存储访问、同步机制、软件栈、测试验证、性能分析和高效仿真等大量系统问题。这些成本会反复出现在不同项目中，显著提高 DSA 从设计到可用的门槛。Buckyball 希望把这些共性的系统能力沉淀为一个可复用、可扩展的开源主线，让研究者和开发者可以更专注于自己的 DSA 设计，同时更容易把它发布出来、接入系统，并与其他加速器协同工作

希望Buckyball可以帮助你更加快速，更高质量地构建自己的DSA项目，共享基础设施。

### 环境构建

无论如何，我们得先让 buckyball 跑起来。


> 💡 准备事项（唯一要求）：任意linux系统+确保能够科学上网（稳定访问外网很重要！）

下面让我们来一键安装

1. 安装Nix。这是个用来管理我们环境的工具，它保证了这个仓库无论三个月还是十年后安装的环境都会一致。

这步希望你有sudo权限，也是唯一一步。

```bash
sudo curl -fsSL https://install.determinate.systems/nix | sh -s -- install
```

当你看到这个表明Nix安装完成

![image.png|697](image1-1.png)


1. 安装仓库

```bash
git clone https://github.com/DangoSys/buckyball.git
cd buckyball
./scripts/nix/build-all.sh
```

当你看到这个表明仓库安装完成啦

![image.png](image1-2.png)

1. 验证安装完成，可以通过执行下面这条命令

```bash
bbdev bebop-verilator --run '--binary ctest_vecunit_matmul_ones_singlecore-baremetal --batch --config sims.verilator.BuckyballToyVerilatorConfig --itrace --mtrace --pmctrace --ctrace --banktrace'
```

跑通显示如下说明仓库安装完成啦（第一次运行会进行安装，所以可能会有点慢）。

![image.png](image1-3.png)

执行的这个是调用向量加速单元执行的矩阵乘，buckyball的旅途从这里开始。
